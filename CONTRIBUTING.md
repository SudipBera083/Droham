# Contributing to Droham

We are delighted that you are interested in contributing to Droham! As an enterprise-grade platform, we maintain high standards for code quality and security.

## Development Process

1.  **Fork the Repository**: Create a personal fork of the project.
2.  **Create a Feature Branch**: Work on a descriptively named branch (e.g., `feature/analytics-upgrade`).
3.  **Adhere to Standards**: Follow PEP 8 for Python and our internal design system for CSS.
4.  **Submit a PR**: Provide a clear description of your changes and reference any related issues.

## Security Vulnerabilities

If you discover a security vulnerability, please do **not** open an issue. Instead, email our security team at [security@droham.com](mailto:security@droham.com).
